Certificat SSL automatique pour sécuriser votre site. Pas besoin de configuration manuelle.
CDN Mondial

Domaine Custom Gratuit

100% Gratuit
Pour les projets personnels comme le vôtre, Vercel est totalement gratuit. Aucune carte de crédit requise.

⚠️ NE PAS Modifier /out Directement

IMPORTANT : Les fichiers HTML dans le dossier /out sont générés automatiquement par Next.js lors du build.
Pourquoi ne pas modifier /out ?
À chaque fois que vous exécutez npm run build, Next.js écrase complètement le dossier /out avec de nouveaux fichiers générés depuis vos fichiers source.
Que se passe-t-il si vous modifiez /out ?
Vos modifications seront perdues au prochain build. C'est du travail inutile.
✅ Solution : Modifiez toujours les fichiers source dans /src, puis reconstruisez avec npm run build.

🎨 Comment Personnaliser l'Apparence

📂 Structure des Fichiers à Modifier

src/ ├── app/ │ ├── page.tsx ← Page d'accueil │ ├── map/page.tsx ← Page carte │ ├── events/page.tsx ← Page événements │ ├── weather/page.tsx ← Page météo │ ├── community/page.tsx ← Page communauté │ ├── admin/page.tsx ← Page admin │ └── ... ├── components/ │ ├── layout/ │ │ └── header.tsx ← Menu de navigation │ └── ui/ ← Composants visuels └── app/globals.css ← Styles globaux

🎯 Fichiers Clés pour l'Apparence

globals.css - Couleurs, polices, styles généraux
Fichier : src/app/globals.css
tailwind.config.ts - Configuration des couleurs, thème
Fichier : tailwind.config.ts
header.tsx - Menu de navigation, logo
Fichier : src/components/layout/header.tsx
page.tsx - Contenu de chaque page
Fichiers : src/app/*/page.tsx

📋 Prochaines Étapes - Améliorer l'Apparence

Personnaliser les Couleurs
Modifier tailwind.config.ts pour changer les couleurs primaires et secondaires du site.
Améliorer la Page d'Accueil
Modifier src/app/page.tsx pour ajouter des images, modifier les textes, améliorer la mise en page.
Personnaliser le Header
Modifier src/components/layout/header.tsx pour changer le menu, ajouter votre logo.
Ajouter des Images
Placer vos images dans public/images/ et les référencer dans vos composants.
Tester en Local
Exécuter npm run dev pour voir vos modifications en temps réel.
Déployer sur Vercel
Exécuter vercel --prod pour mettre à jour le site en ligne.

🔄 Workflow de Développement

1. Développement Local
npm run dev - Démarre le serveur de développement sur http://localhost:3000
2. Modifier les Fichiers Source
Éditez les fichiers dans /src et voyez les changements instantanément dans votre navigateur.
3. Tester en Local
Vérifiez que tout fonctionne correctement avant de déployer.
4. Build de Production
npm run build - Génère les fichiers optimisés dans /out
5. Déploiement
vercel --prod - Déploie automatiquement sur Vercel et met à jour votre site en ligne.
✅ Résultat : Votre site est mis à jour automatiquement et accessible mondialement en 2-3 minutes.
